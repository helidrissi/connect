Changes 0.81.3

There are three basic changes in versions JRecord 0.81.2/0.81.3:

  • Support for Complex Occurs Depending.
  • Simplified Xml Schema Definition
  • Extra JRecord Options for processing Xml schema's

Occurs Depending Changes

These changes are intended to support nested Occurs Depending Size variables. i.e 

4                03  size-1                pic s9(4).
5                03  occurs 1 to 20 times depending on size-1.
6                    05  size-11           pic s9(4).
7                    05  occurs 1 to 20 times depending on size-11.

Xml Schema Changes

Changes in the latest version of JRecord reduces the number of options that need to be coded in a Xml-Schema

Traditionally with Xml-Schema you would specify both position and length: 

 1 <?xml version="1.0" ?>  
 2 <RECORD RECORDNAME="DTAR020" FONTNAME="CP037" FILESTRUCTURE="Default" RECORDTYPE="RecordLayout">
 3         <FIELDS>
 4                 <FIELD NAME="KEYCODE-NO" POSITION="1"  LENGTH="8" TYPE="Char" />
 5                 <FIELD NAME="STORE-NO"   POSITION="9"  LENGTH="2" TYPE="Mainframe Packed Decimal (comp-3)" />
 6                 <FIELD NAME="DATE"       POSITION="11" LENGTH="4" TYPE="Mainframe Packed Decimal (comp-3)" />
 7                 <FIELD NAME="DEPT-NO"    POSITION="15" LENGTH="2" TYPE="Mainframe Packed Decimal (comp-3)" />
 8                 <FIELD NAME="QTY-SOLD"   POSITION="17" LENGTH="5" TYPE="Mainframe Packed Decimal (comp-3)" />
 9                 <FIELD NAME="SALE-PRICE" POSITION="22" LENGTH="6" DECIMAL="2" TYPE="Mainframe Packed Decimal (comp-3)" />
10         </FIELDS>
11 </RECORD>

Now you can just specify the length and the position will be calculated: 

 1 <?xml version="1.0" ?>
 2 <RECORD RECORDNAME="DTAR020" FONTNAME="CP037" FILESTRUCTURE="Default" RECORDTYPE="RecordLayout">
 3         <FIELDS>
 4                 <FIELD NAME="KEYCODE-NO"  LENGTH="8" TYPE="Char" />
 5                 <FIELD NAME="STORE-NO"    LENGTH="2" TYPE="Mainframe Packed Decimal (comp-3)" />
 6                 <FIELD NAME="DATE"        LENGTH="4" TYPE="Mainframe Packed Decimal (comp-3)" />
 7                 <FIELD NAME="DEPT-NO"     LENGTH="2" TYPE="Mainframe Packed Decimal (comp-3)" />
 8                 <FIELD NAME="QTY-SOLD"    LENGTH="5" TYPE="Mainframe Packed Decimal (comp-3)" />
 9                 <FIELD NAME="SALE-PRICE"  LENGTH="6" DECIMAL="2" TYPE="Mainframe Packed Decimal (comp-3)" />
10         </FIELDS>
11 </RECORD>

Or you can specify the position and the length will be calculated. You must still specify the length for the last field though: 

   1 ?xml version="1.0" ?>
   2 <RECORD RECORDNAME="DTAR020" FONTNAME="CP037" FILESTRUCTURE="Default" RECORDTYPE="RecordLayout">
   3         <FIELDS>
   4                 <FIELD NAME="KEYCODE-NO" POSITION="1"    TYPE="Char" />
   5                 <FIELD NAME="STORE-NO"   POSITION="9"    TYPE="Mainframe Packed Decimal (comp-3)" />
   6                 <FIELD NAME="DATE"       POSITION="11"   TYPE="Mainframe Packed Decimal (comp-3)" />
   7                 <FIELD NAME="DEPT-NO"    POSITION="15"   TYPE="Mainframe Packed Decimal (comp-3)" />
   8                 <FIELD NAME="QTY-SOLD"   POSITION="17"   TYPE="Mainframe Packed Decimal (comp-3)" />
   9                 <FIELD NAME="SALE-PRICE" POSITION="22" LENGTH="6" DECIMAL="2" TYPE="Mainframe Packed Decimal (comp-3)" />
  10         </FIELDS>
  11 </RECORD>

For Csv Files, you can do: 

   1 <?xml version="1.0" encoding="UTF-8"?>
   2 <RECORD RECORDNAME="CsvTst" DELIMITER="comma" FILESTRUCTURE="CSV_NAME_1ST_LINE" STYLE="0" RECORDTYPE="Delimited" QUOTE="">
   3         <FIELDS>
   4                 <FIELD NAME="f1"/>
   5                 <FIELD NAME="f2"/>            
   6                 <FIELD NAME="f3"/>
   7                 <FIELD NAME="f4"/>
   8                 <FIELD NAME="f5"/>
   9         </FIELDS>
  10 </RECORD>

JRecord changes for Xml schema's

There are a number of small changes to JRecord

  • When loading Xml schema's the font (Character-set) is now a default value used if the font is not specified in the Xml-Schema.
  • For ExternalRecord class, the setFontName returns the ExternalRecord, this allows it to be used in "fluid style" programming
  • A asIOBuilder() has been added to the ExternalRecord class.
  • a new setDefaultFont has been added to the SCHEMA_XML iobuilder. This method sets a default font to be used if a Font is not specified in the Xml-Schema.
  • The setFont method of the SCHEMA_XML iobuilder will set the font no matter what is specified in the Xml.

    • Now when you specify a defaultfont, when loading a Xml-Schema, it becomes the default font (if non is specified in the Xml): 

    14         RecordEditorXmlLoader loader = new RecordEditorXmlLoader();
15         ExternalRecord xrec = loader.loadCopyBook("Schema.Xml", 0, 0, "CP037", 0, 0, 0, null);
    or using IOBuilders, you use the setDefaultFont method 
    16         AbstractLineReader reader  = JRecordInterface1.SCHEMA_XML
17                                         .newIOBuilder(copybookName)
18                                             .setDefaultFont("cp037")
19                                         .newReader("InputFile.txt");

    You can overide the font in the Xml with the setFontName method of ExternalRecord class: 

    18         RecordEditorXmlLoader loader = new RecordEditorXmlLoader();
19         ExternalRecord xrec = loader.loadCopyBook("Schema.Xml", 0, 0, "", 0, 0, 0, null)
20                                     .setFontName("CP037");
    or using IOBuilders, you use the setFont method to overide the font in the Xml: 
    16         AbstractLineReader reader  = JRecordInterface1.SCHEMA_XML
17                                         .newIOBuilder(copybookName)
18                                             .setFont("cp037")
19                                         .newReader("InputFile.txt");

    You can combine this with the asIOBuilder() method: 

    30         RecordEditorXmlLoader loader = new RecordEditorXmlLoader();
31         AbstractLineReader reader = loader.loadCopyBook("Schema.Xml", 0, 0, "", 0, 0, 0, null)
32                     .setFontName("CP037")
33                     .asIOBuilder()
34                         .newReader("FileName.txt");

    or 

    23          AbstractLineReader reader = JRecordInterface1.SCHEMA_XML
24                 .newIOBuilder("Schema.Xml")
25                 .getExternalRecord()
26                     .setFontName("cp1252")
27                     .asIOBuilder()
28                         .newReader("FileName.txt");
