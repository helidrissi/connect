Changes

Extended Occurs Depending support

There is now support for complex Occurs Depending: 

4                03  size-1                pic s9(4).
5                03  occurs 1 to 20 times depending on size-1.
6                    05  size-11           pic s9(4).
7                    05  occurs 1 to 20 times depending on size-11.

JRecord changes for Xml schema's

There are a number of small changes to JRecord

  • When loading Xml schema's the font (Character-set) is now a default value used if the font is not specified in the Xml-Schema.
  • For ExternalRecord class, the setFontName returns the ExternalRecord, this allows it to be used in "fluid style" programming
  • A asIOBuilder() has been added to the ExternalRecord class.
  • a new setDefaultFont has been added to the SCHEMA_XML iobuilder. This method sets a default font to be used if a Font is not specified in the Xml-Schema.
  • The setFont method of the SCHEMA_XML iobuilder will set the font no matter what is specified in the Xml.

    • Now when you specify a defaultfont, when loading a Xml-Schema, it becomes the default font (if none is specified in the Xml): 

    14         RecordEditorXmlLoader loader = new RecordEditorXmlLoader();
15         ExternalRecord xrec = loader.loadCopyBook("Schema.Xml", 0, 0, "CP037", 0, 0, 0, null);
    or using IOBuilders, you use the setDefaultFont method 
    16         AbstractLineReader reader  = JRecordInterface1.SCHEMA_XML
17                                         .newIOBuilder(copybookName)
18                                             .setDefaultFont("cp037")
19                                         .newReader("InputFile.txt");

    You can overide the font in the Xml with the setFontName method of ExternalRecord class: 

    18         RecordEditorXmlLoader loader = new RecordEditorXmlLoader();
19         ExternalRecord xrec = loader.loadCopyBook("Schema.Xml", 0, 0, "", 0, 0, 0, null)
20                                     .setFontName("CP037");
    or using IOBuilders, you use the setFont method to overide the font in the Xml: 
    16         AbstractLineReader reader  = JRecordInterface1.SCHEMA_XML
17                                         .newIOBuilder(copybookName)
18                                             .setFont("cp037")
19                                         .newReader("InputFile.txt");

    You can combine this with the asIOBuilder() method: 

    30         RecordEditorXmlLoader loader = new RecordEditorXmlLoader();
31         AbstractLineReader reader = loader.loadCopyBook("Schema.Xml", 0, 0, "", 0, 0, 0, null)
32                     .setFontName("CP037")
33                     .asIOBuilder()
34                         .newReader("FileName.txt");

    or 

    23          AbstractLineReader reader = JRecordInterface1.SCHEMA_XML
24                 .newIOBuilder("Schema.Xml")
25                 .getExternalRecord()
26                     .setFontName("cp1252")
27                     .asIOBuilder()
28                         .newReader("FileName.txt");

    Setting Record Starting position

    With the Multi-Copybook loader you can now set the Records starting position either by absolute position:

    20         ICobolMultiCopybookIOBuilder iob = JRecordInterface1.COBOL
21                 .newMultiCopybookIOBuilder("MultiCopybook")
22                     .setDialect(ICopybookDialects.FMT_MAINFRAME)
23                     .addCopyBook("DTAR020.cbl")
24                     .addCopyBook("SecondRecord.cbl"))
25                         .setStartingPosition(120);

    Or by specifying a field:

    28         ICobolMultiCopybookIOBuilder iob = JRecordInterface1.COBOL
29                 .newMultiCopybookIOBuilder("MultiCopybook")
30                     .setDialect(ICopybookDialects.FMT_MAINFRAME)
31                     .addCopyBook("Master.cbl")
32                     .addCopyBook("Child1.cbl")
33                         .setStartingPositionToField("Master.cbl", "Record-Detail")

    This is to handle the case where you have a Master copybook like:

    5        01  Master.
6            03 Header-Details .
7               ...
8            03 Record-Details                          Pic x(300).

    And you move a series of Child copybooks to the Record-Details field.

    Insert Field in External Record

    A new method addFixedWidthField has been added to the ExternalRecord class. This method has 2 parameters:

    1. field Field to be inserted
    2. Position-adjustment - how following fields are to be adjusted. Possible values
      • NO_ADJUSTMENT - no adjustment to the position of other fields is made
      • ADJUST_BY_LENGTH - Simply add length of the new field to the position of the following fields.
      • CALCULATE_ADJUSTMENT - Calculate a Position adjustment adjustment 
     8             ExternalRecord.FieldAdjustmentOptions adj 
 9                     = ExternalRecord.FieldAdjustmentOptions.ADJUST_BY_LENGTH;
10             ExternalField field = new ExternalField(
11                       POSITIONS[i] + adjAmount, 5, "name", "", Type.ftChar,
12                       0, 0, "", "", "", 0);
13             int idx = externalRec.addFixedWidthField(field, adj);

    Which adjustment to use 

      If you have
   
      <field_1><field_2>

  and you want to use add a new field
  
     <field_1><new_field><field_2>
  
  you might as well use ADJUST_BY_LENGTH but CALCULATE_ADJUSTMENT will also work
  
  If field to be inserted partially overlaps a filler / existing field
  
     <filler ------><field_1>
     
  and you want to change to
  
     <filler ----<new_Field --><field_1>
     
  use CALCULATE_ADJUSTMENT
